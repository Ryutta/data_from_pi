# Rでの分析をPower BIで実装するためのガイド

これまでR（`analysis_workflow.Rmd`）で実装していたデータの取得、整形、U値計算、およびキャンペーンごとの評価・可視化をPower BI上で実現するための手順を解説します。

## 1. データの取得と整形 (Power Query)

Rでの `httr` および `jsonlite` を用いたAPIアクセスとデータ展開は、Power BIの **Power Query (M言語)** を用いて同等の処理が可能です。

* **データソースの接続**:
  すでに `data_from_pi.SemanticModel/definition/expressions.tmdl` に設定されているパラメータ（`PI_System_URL`, `PI_Server_Name`, `PI_Tag` など）を使用してPI Web APIからデータを取得します。
* **JSONデータの展開**:
  取得したJSONレスポンスの `Items` リストを展開し、`Timestamp` および `Value` 列を抽出します。`Value` が入れ子になっている場合（例: Digital State）は、Rで実装していたように、内部の `Value` またはリストの最初の要素を抽出するカスタム列をPower Queryで追加します。
* **タイムゾーン変換と型変換**:
  `Timestamp` をUTCから日本時間（+09:00）に変換し、データ型を「日時（Date/Time）」または「日付（Date）」に設定します。`Value` は「10進数（Decimal Number）」に変換します。
* **横持ち（Pivot）への変換**:
  U値の計算を容易にするため、`Timestamp` を行、`Tag` を列、`Value` を値としてピボット解除（Pivot）し、各タグ（`M2FC4403.PV`, `M2PI4410W.PV` など）を個別の列として保持するテーブルを作成します。

## 2. 総括伝熱係数 (U値) の算出 (DAX または Power Query)

U値の算出は、Power Queryのカスタム列として追加するか、DAXの計算列またはメジャーとして実装します。計算のパフォーマンスと柔軟性を考慮すると、DAXの計算列が適しています。

`U_Calculation_Method.md` に定義されている計算式をDAXで表現します。

```dax
// ポンプ流量の計算例 (計算列)
Pump_Inlet = ('データ'[M2PI4407.PV] / 760 * 101.325) + (950 * 9.81 * (1.4 + 6.125) / 1000)
Pump_Outlet = 'データ'[M2PI4411W.PV] + 101.325
Lifting_Height = (Pump_Outlet - Pump_Inlet) * 1000 / 9.81 / 950
Pump_Flow = (-0.6857 * Lifting_Height^2 + 0.8571 * Lifting_Height + 43.589) * 60 * 950 / 1000

// 対数平均温度差 (LMTD) と U値 の計算例 (計算列)
t3_t4 = 27.838 * LN('データ'[M2PI4410W.PV] + 101.325) - 28.559
dT1 = t3_t4 - 'データ'[M2TI4405.PV]
dT2 = t3_t4 - t2 // t2 は前段で計算
delta_tlm = DIVIDE((dT1 - dT2), LN(dT1 / dT2))

U_Value = DIVIDE(('データ'[q] * 1000 * 1000 / 367), delta_tlm)
```

## 3. キャンペーン評価用のデータ準備 (Power Query / データモデリング)

`campaign_periods.csv` をPower BIに読み込みます。

* **期間の特定**:
  Power Queryで「開始から1ヶ月（初期）」および「終了前1ヶ月（末期）」の日付範囲を計算する列を追加します。
* **最大値の抽出**:
  DAXの `CALCULATE` 関数と `MAXX` などを組み合わせるか、Power Query側で各期間（初期・末期）ごとの `M2PI4410W.PV`（ジャケット圧力）の最大値となる行をフィルタリングして、抽出用のサマリーテーブルを作成します。
* **データのリレーションシップ**:
  キャンペーンの評価テーブル（サマリーテーブル）と `campaign_periods.csv` を、作成したキー（例: `2310_Beginning`）を介してリレーション（1対1または多対1）で結合し、詰まり本数（NoS）のデータと紐づけます。

## 4. ビジュアルの作成 (Power BI レポート)

Rで描画していたグラフや表は、以下の標準ビジュアルで再現できます。

1. **PIデータトレンド (折れ線グラフ)**
   * **X軸**: `Timestamp`
   * **Y軸**: `Value` (タグごとに表示する場合はピボット前のデータを使用し、`凡例` に `Tag` を設定)
   * **機能**: ズームスライダーを有効にすると、詳細な期間の傾向が確認しやすくなります。

2. **U値および関連パラメータの表 (テーブル / マトリックス)**
   * テーブルビジュアルを使用し、`Timestamp`, `M2PI4410W`, `q`, `pump_flow`, `U_Value` などの列を配置します。条件付き書式を利用して、U値が低下している箇所を色付けすることも可能です。

3. **相関分析の散布図 (散布図ビジュアル)**
   * **X軸**: `Max_Jacket_P`, `U_Value`, または `Flow_Rate`
   * **Y軸**: `詰まり本数 (NoS)`
   * **凡例**: `Type` (Beginning / End)
   * **機能（傾向線とR二乗値）**:
     ビジュアルの「分析」ペイン（虫眼鏡アイコン）から **「傾向線（Trend line）」** を追加します。さらに、R言語のグラフのように回帰式やR²（決定係数）を表示したい場合は、以下のいずれかの方法を取ります：
     * **DAXでの算出**: `LINEST` 関数（新しいDAX関数）を利用して、回帰係数やR²を計算するメジャーを作成し、カードビジュアルやツールチップで表示する。
     * **Python / R ビジュアルの利用**: Power BIの標準機能で「Rスクリプトビジュアル」または「Pythonビジュアル」を選択し、Rで作成した `ggplot2` のコードをそのまま移植して描画する。

4. **予測モデル評価結果の表 (テーブル)**
   * Power BI上でDAXを用いて `LINEST` 関数で予測モデルを構築するか、事前に予測値を計算したデータテーブルを読み込み、実際の値との誤差（MAPE等）を計算します。
   * 条件付き書式（データバーまたは背景色）を設定し、誤差率が10%未満の場合は緑、50%以上の場合は赤などの「評価結果」を視覚的に強調表示させます。
